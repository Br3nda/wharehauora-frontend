Pattern Library: Forms

Text Input

  • (helper text)

All form fields should be contained within a ul.form-row with each label and input pair in their own li to give the table it's formatting structure.

Form labels may be proceeded by helper text using span.helper immediately after a label.

To use two side-by-side half-width form fields, apply class half to the parent li.
To restrict a form field to half width even when it's the only field on the row, apply class max-half to the parent li.
To add additional whitespace above a label/input pair, apply class space to the parent li.

To add error styling to any label/input pair, add class error to the parent li.

Example

<ul class="form-row"> <li> <label>Text input</label><span class="helper">(helper text)</span> <input type="text" /> </li> <li class="half"> <label>Side by...</label> <input type="text" /> </li> <li class="half"> <label>side inputs</label> <input type="text" /> </li> <li class="max-half"> <label>Input restricted to half width</label> <input type="text" /> </li> <li> <label>Text input w/ placeholder</label> <input type="text" placeholder="Placeholder" /> </li> <li> <label>Text input w/ value</label> <input type="text" value="Value" /> </li> <li> <label>Text input disabled</label> <input type="text" value="Value" disabled /> </li> <li class="error"> <label>Text input error message</label> <input type="text" value="Value" /> </li> <li class="space"> <label>Text input w/ space above</label> <input type="text" /> </li> </ul>

Textarea

Textarea as per standard HTML tag use. To add error styling to any label/textarea pair, add class error to the parent li.

Select

Select boxes use SelectBoxIt.js to replace with styleable custom controls. No additional markup is required for this styling. The first option will be hidden from the options list, so use it as instruction/placeholder, not as a selectable value. All other size/position options from text inputs are available for select boxes as well, including the same error styling.

Example

<select> <option value="">Please select</option> <option value="value1">Option One</option> <option value="value2">Option Two</option> <option value="value3">Option Three</option> <option value="value4">Option Four</option> <option value="value5">Option Five</option> <option value="value6">Option Six</option> <optgroup label="Group Heading"> <option value="value1">Option One</option> <option value="value2">Option Two</option> <option value="value3">Option Three</option> <option value="value4">Option Four</option> </optgroup> </select>

Checkbox & Radio

Lists of checkboxes or radio button options are contained within their own ul.form-controls list. Use this parent list even if there is only a single checkbox. Always put the input inside its label, with no space between input tag and label text.

Checkboxes and radio buttons use icheck.js to replace with custom styleable controls. No additional markup is required for this styling.

Example

<ul class="form-row"> <li> <label>Checkbox list</label> <ul class="form-controls"> <li><label><input type="checkbox" name="group" value="value1" />Option One</label></li> <li><label><input type="checkbox" name="group" value="value2" checked />Option Two</label></li> <li><label><input type="checkbox" name="group" value="value3" disabled />Option Three</label></li> <li><label><input type="checkbox" name="group" value="value4" checked disabled />Option Four</label></li> </ul> </li> <li> <label>Radio list</label> <ul class="form-controls"> <li><label><input type="radio" name="group2" value="value1" />Option One</label></li> <li><label><input type="radio" name="group2" value="value2" />Option Two</label></li> <li><label><input type="radio" name="group2" value="value3" disabled />Option Three</label></li> <li><label><input type="radio" name="group2" value="value4" checked disabled />Option Four</label></li> </ul> </li> </ul>

Action Cancel

Except for rare occasions – like pre-login forms and inline form – form submit or cancel actions should always be placed within the card or modal footer. Add class submit to the a.btn to make it submit its parent form on click.

Important note: If a card or modal is a form, change the card/modal HTML element to form.card instead of section.card or form.modal instead of div.modal. If a form is contained within div.inset you can make that form.inset as the form container. Or, add a separate form element around ul.form-row if there is no suitable parent container to turn into the form.

Example

<form class="card"> <header> <h2>Card Title</h2> </header> <p>Form here</p> <footer> <a class="btn submit" href="#">Submit</a> <a class="del" href="#"><img class="svg" src="./img/icon/del.svg" alt="icon" />Cancel</a> </footer> </form>

Inline Form

  • Submit

Inline forms are only used for very simple forms that act as creators of new list/table rows (e.g. admin creating new room type). If there are too many fields to use an inline form, put the form in a modal instead. Add class inline to ul.form-row to allow for this form layout.

Inline forms may only be composed of a single li.half input, with the optional addition of no more than two li.small inputs (which are even narrower). This small input size is only available for inline forms.

Example

<form class="inset"> <ul class="form-row inline"> <li class="half"> <label>Label</label> <input type="text" /> </li> <li class="small"> <label>Label</label> <input type="text" /> </li> <li class="small"> <label>Label</label> <input type="text" /> </li> <li class="button"> <a class="btn submit" href="#">Submit</a> </li> </ul> </form>
