One or more .reading's are always contained within a parent .readings container.

All readings follow the same basic structure of visualisation (colour and icon), value, label, and time.

Example

<div class="reading temp-mid-a"> <span class="vis"><span class="loader"></span></span> <span class="value">22.5°C</span> <span class="label">Temperature</span> <span class="time">5 minutes ago</span> </div>

Temperature colour and icon (scale of 10) is determined by the class name on .reading as follows:

temp-high-2b Very hot
temp-high-2a Hot
temp-high-1b Very warm
temp-high-1a Warm
temp-mid-b Comfortable warm
temp-mid-a Comfortable cool
temp-low-1a Cool
temp-low-1b Very cool
temp-low-2a Cold
temp-low-2b Very cold

Expired & Loading

22.5°C Temperature 8 hours ago
00.0°C Temperature Loading...

Add class expired to any .reading who's latest reading is too old to be accurate. This will create the light grey colouring.

Example

<div class="reading temp-mid-a expired"> <span class="vis"><span class="loader"></span></span> <span class="value">22.5°C</span> <span class="label">Temperature</span> <span class="time">8 hours ago</span> </div>

Add class loading to any .reading (or the reading's parent container/card) while the value is loading. At the same time the value should be 00.0 and the "time since" should be Loading.... Once the data is received, insert the correct value and time, and then remove the loading class. Loading animation will display only while loading class is applied.

Example

<div class="reading temp-mid-a loading"> <span class="vis"><span class="loader"></span></span> <span class="value">00.0°C</span> <span class="label">Temperature</span> <span class="time">Loading...</span> </div>
